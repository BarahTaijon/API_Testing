Variables

- what? it is where we can store some data.
- why needed in postman? to avoid duplication
- Where to create variable? created in multiple Level/Scope of variables [based on the scope access lvl]: Global, Collection, Environment, Local, data
- How to create? (in the corner) click on the icon called `variables in this reqest`.


1. Global
- can access through in the workspace [all collections, folders, and requests]
- to add one:
    1. click here `variables in this reqest` icon
    2. click *add* under Global section
    3. add the name & value of the variable 
- add th var to all Req then open the consol to see the var part.


Note: - var is sth common (e.g., Host in the URL).
      - Don't forget to **save** after adding variables 

2. collection
- can access only within collection
- to add one:
    1. click here `variables in this reqest` icon
    2. click *add* under (the name of the collection) section
    3. add the name & value of the variable 


3. Environment
- accessable in all collections within a particular environment.
- Is a named set of key-value pairs (variables) that allows you to manage different setups.
  environment: Development, Staging/testing, pre-production, and production.
- change URLs & values - without changing your API requests.
- to add one:
    1. click here `variables in this reqest` icon
    2. Add/click on an environment
    3. click *add* under Environment section
    4. add the name & value of the variable 

Note: Env is Globle within a particular env


4. Local
- accessible only within request.
- created on request level
- Can only defined in `pre-reqest script` 
- to add one:
    1.  on `pre-reqest script` 
    2. define the name & value of the var. ex.:
	//loacal variable
	pm.variables.set("url_local","http://localhost:3000");
