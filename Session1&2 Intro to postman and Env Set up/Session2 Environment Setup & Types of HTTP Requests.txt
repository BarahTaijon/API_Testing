Workspace & Collection & Requests




Workspace
- The Area where we maintain and save files [test cases, requests, …].
- collaborative, secure environments to organize, share, and manage APIs, collections, and environments.
- enable team collaboration (private or team), personal development, and public API sharing.

1. create: Users can create them via the Workspaces header dropdown, select visibility, and track changes.
Workspaces → Create Workspaces 

2. delete: Users can delete them via the Workspaces via the settings tab
setting tab → Delete Workspaces 

3. rename:
Just double-click on the Workspace name




Collection
- groups of saved API requests that allow you to organize, document, share, and automate your API testing workflows.
- support nesting folders, shared variables, authorization settings, and scripts across requests, enabling efficient collaboration and API lifecycle management. 
- contain number of folders and http requests.
- under workspace we can create set of collections

1. create
Collections → + → blank Collection
Collections → New → Collection

2. delete 
on the same name of the collection [three dotes] → click → delete

3. rename:
just double click on collection name

Requests
- Query Parameters and Path Parameters. 
- used to send specific instructions or data to a server within the URL. 
-HTTP Requests Types
	Get - retrieve a recourse from database
	Post - create a resource on database 
	Put - update existing resource on database
	Patch - for partial updating  
	Delete - delete existing resource form database

1. create
on the same name of the collection [three dotes] → click → Add a req

2. delete 
three dotes

3. rename
duble click on Req name




Response 
- the validation is done on the Response Payload part, is 
	- Status validation
	- body/payload [.json or .xml]. 
	- cookies 
	- Headers
	- Test Results
	- we get also information [time & size of data]


- Note:
    - when you send the request in the PUT & POST, we put the ID in the URL :
		ex: https://reqres.in/api/users/673
    
    - when you send the request in the GET : 
        	ex: https://reqres.in/api/users?page=2
		    https://reqres.in/api/users?page=3
        
    - at the end of writing your request click **save** to save your work

    - to run all the requests - three dots on collection name → Run Collection
