# Upload file/s in Postman
	

	- Write cmd in the path of the jar file "java -jar file-upload-RestAPI.jar". Jar now running 

	- In browser write : http://localhost:8080/ 	
	    →  On this port number your API will run, and you can see UI app 
	    →  here you can UI functionality, that's should hapen through API


- POST {{UploadLink}}/uploadFile 	or 	http://localhost:8080/uploadFile

	1. Go to Body section, under it, Go to form-data
	2. Under `Key` write : file , in dropdown choose File (instead of Text)
	3. Under `Value` specifiy the file you want to upload
		Note: Make sure that the uploaded file is in the location specified by Postman 
		      [ to know the loc go to setting → (ctrl+ , )/General/loaction ]



- POST {{UploadLink}}//uploadMultipleFiles 	or 	http://localhost:8080//uploadMultipleFiles

	
	Same steps but in `Key` write : files

# Authentication types

1. API Key - we want a key
2. Bearer token - we want a token
3. Basic Auth - need password & username - data not encrypted
4. Digest Auth - need password & username - data encrypted [more secure]
4. OAuth 1.0 - third party auth - need third party code
5. OAuth 2.0 - third party auth


Note: we can't use every type of auth for every type of API,
      It depends on how the api got designed, based on that we used particuler auth.
      In requirement, it'ill be provide u what type of auth u can use


=====================================================


Basic Auth
The endpoint accepts a default username and password and returns a status code of 200 ok only if the same is provided.
Otherwise it will return a status code 401 unauthorized.

# Swagger
 - is an interactive documentation
 [if dev design some api's, if he want to provide some info about these api's
  so he want to document it using Ms word or pdf, these doc aren't interactive docs]
 [dev share info about api's what is the reqest url, what kind of request u have to send , kind of response you 
  will get into interactive doc called swagger]
 
 - the difference between static & interactive : that in interactive doc itself you can explore API's 
	that will interact with the server 

 - before testing the actual API's, we should go to swagger document because it contains a detailed info about ur api
	if it's only info you just read it
	if it's an interactive info u try itf

 - Look at this example: https://fakerestapi.azurewebsites.net/index.html
	( click try it & execute ) 
	- try it on postman to ensure that it gives same response

 - Difference betw Postman & Swagger - there are many differences the main diff
	* swagger used to only for exploring the api's not for testing purpose
	  postman we do the testing.
	* In swagger, we cann't put any validations or tests, in postman we can
	* In swagger, we cann't save results anywhere else




# cURL
 - stands for Client URL
 - it is a command (you can try it in the cmd)
 - stime in API doc a curl is provided 
 - some doc, don't directly provide the request url,
	instead of that, they gives curl
 - curl contains url along with the headers, information, authenticaion
 - instead of giving all info (in body & auth & param sections), you only give the crul 

 - To creat a request form a curl: copy curl &  click import  & paste it
