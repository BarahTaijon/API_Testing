- API chaining:
	 is when you send an API request, and receive a response,
	 then send that response as a request to the next API
	 (So the response of one API became a request in another API)
- There will be some dependencies between requests.
- here, you need to store the response in a variable, then use it in the reqest of another API

Example
1. First, in POST request create a new student without send id then capture the id in the Post-request 

    var jsonData = JSON.parse(responseBody);
    pm.environment.set("student_id", jsonData.id);

    console.log(pm.variables.get("user_id"));

2. Then in the GET & DEL requests, just add the created variable to the link.

   http://localhost:3000/students/{{student_id}}

============

Remote API: no need to make setup/json, only send the url through postman

Local API: run json server, and to have a json file 

Authorization: is a prerequisite to access the Remote API (through inernet) 
	- Request methods (PUT, POST, PATCH, DELETE) needs access token, which needs to be passed with "Authorization" header as Bearer token
	- Authorization tokens has many types (wil discuss later).
        - One of these types is "Bearer token". 
		- so we want to create it.


=============


https://gorest.co.in/  → is a website for sample API with end points for explore & testing purposes


*End Points:

url: https://gorest.co.in/

end point:
POST		/public/v2/users		Create a new user
GET		/public/v2/users/7984698	Get user details
PUT|PATCH	/public/v2/users/7984698	Update user details
DELETE		/public/v2/users/7984698	Delete user


*Token: In GoRESRT, click `Get your access token` to get one.

*Request Body: you should know the Request Body, under `Create User` 

{

}

Ex
-> now apply them in a chaining:
	1.create a user
	2. from id, 
	    2.1 get the same user and verify same data
	    2.2 update the user details
	    2.3 delete user 

First,  create the required requests 
Second, put the token in the `Authorization` [author tab bside body & scripts tabs]
	request lvl (req)→ Authorization → Bearer Token →  put the token
	collection lvl (col)→ Authorization → Bearer Token → put the token → (req)→ Authorization → inherit auth from parent
Third,  we want to send dynamic data (body is static, and email is unique for every user), so to create a dynamic data,
	
	in pre-scripts:

	var random = Math.random().toString(36).substring(2);

	var email = "Barah." + random + "@15ce.com";
	var username = "Barah" + random;

	pm.environment.set("created_email", email);
	pm.environment.set("created_username", username);

	in body:

	{
	"name":"{{created_username}}", 
	"gender":"female", 
	"email":"{{created_email}}",
	"status":"active"
	}


Fourth, Same in the PUT to update a user dynamically, we should create random in script/ send it in body

	in pre-scripts:

	var random = Math.random().toString(36).substring(2);

	var email = "Barah." + random + "@15ce.com";
	var username = "Barah" + random;

	pm.environment.set("email", email);
	pm.environment.set("username", username);

	in body:

	{
	"name":"{{username}}", 
	"gender":"female", 
	"email":"{{email}}",
	"status":"active"
	}
	


fifth, we want to compare the GET request info with the original created one (randomly) from the POST.

     - to compare them we should add a test assertion in GET post response


	//validate json fields in the response 

	pm.test("values of json fields", ()=>{

	var jsonData = pm.response.json(); //parse the response body to json
	pm.expect(jsonData.name).to.eql(pm.environment.get("created_username"));
	pm.expect(jsonData.email).to.eql(pm.environment.get("created_email"));

	});
